how it works

Boring on purpose. Deterministic by stage.

Runs unattended on a schedule, or on demand from chat. Every stage is checkpointed and reported — the worst it can do is waste a run.

01

scheduled trigger

wakes on cron, pulls the assigned queue.

02

agent implements

traces the code and writes the change.

03

build & test

compiles, runs the full suite in isolation.

04

verify & guard

confirms tests truly pass; blocks publish.

05

review in chat

posts the diff + log. you approve.

trust model

An agent that writes code is the easy part.

The hard part is making it safe and reliable enough to run while you sleep. This is the scaffolding around the model.

verify, don't trust

build · test · static checks

parses the real build and test output. a model claiming success is not success.

least privilege

no network · no secrets

runs sandboxed — can't reach the network, publish code, or read secrets.

propose, never publish

autonomy stops at the diff

every change is left uncommitted for review. git state is hard-verified, with alerts on drift.

dead-man's switch

silence is a fault

a scheduled health check flags silent failures. a missing result is itself an alarm.

two-way chatops

ask · steer · approve

reports each run and takes new instructions from chat — multi-turn, from anywhere.

self-review loop

definition of done

re-reads its own diff for missing tests, edge cases, and docs before it hands off.
